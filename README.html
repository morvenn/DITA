Oxygen DITA Project with Custom Editing Settings

This is a sample DITA Project which imposes custom settings for DITA editing. When the project is opened in the Oxygen Project view it will impose a custom set of global options. The loaded project will also impose a custom DITA framework extension which customizes DITA editing in the application.

When the project is opened in the Project view, the following global settings are imposed:

  • The Preferences->Spell check page is saved at project level and the automatic spell checking is enabled.

  • The Preferences->Author page is saved at project level and the full tags rendering mode is enabled.

  • The Preferences->Document Type Association / Locations page is saved at project level and the DITA framework customizations which come with this project in the frameworks folder are used.

When creating and editing DITA content the custom framework customization imposes the following settings:

  • If you edit the Preferences->Document Type Association->"DITA Extension" framework configuration, in the Author->Actions tab you can remove or add extra actions. The current DITA framework extension removes the DITA <ul/> underline action from the toolbar. More about custom Author actions: https://blog.oxygenxml.com/topics/custom-author-action-content-completion.html

  • If you edit the Preferences->Document Type Association->"DITA Extension" framework configuration, in the Validation tab you can find a DITA validation scenario. Besides the regular validation steps it adds an extra frameworks/customDITA/rules/rules.sch Schematron file included in the project which shows an error when the DITA <u> underline element is being used. More about Schematron validation: https://blog.oxygenxml.com/topics/sharing_schematron_validation_rules.html

  • If you edit the Preferences->Document Type Association->"DITA Extension" framework configuration, in the Templates tab there are references to custom new file template folders. The frameworks/customDITA/templates/ templates folder included in the project can include custom new file templates. More about new file templates: https://blog.oxygenxml.com/topics/author-document-type-extension-sharing-custom-new-file-templates.html

  • If you edit the Preferences->Document Type Association->"DITA Extension" framework configuration, in the Author->CSS tab there is a reference to an extra custom.css stylesheet. The frameworks/customDITA/css/custom.css stylesheet has a custom CSS rule which makes the DITA <title> element red. More about custom CSS: https://blog.oxygenxml.com/topics/customizeDITACSS.html

  • If you edit the Preferences->Document Type Association->"DITA Extension" framework configuration, in the Classpath tab there is a reference to an extra resources folder located in the project. The frameworks/customDITA/resources/cc_config_ext.xml configuration file rejects the insertion of DITA <b>, <i> and <u> elements. More about custom configuration files: https://www.oxygenxml.com/doc/versions/22.1/ug-editor/topics/configure-elements-attr-cc-individually.html

  • If you edit the Preferences->Document Type Association->"DITA Map Extension" framework configuration, in the Transformation tab there is an extra transformation scenario named DITA Map WebHelp Responsive (508 - accessibility) which refers to a publishing template zip archive. The frameworks/customDITA/publishing/CustomTemplate508.zip publishing template file is used by the transformation scenario. More about publishing templates: https://www.oxygenxml.com/doc/versions/22.1/ug-editor/topics/whr-publishing-template-2.html.

  • The src/userguide.ditamap main DITA Map is added to the Master files folder. This allows Oxygen to automatically update references in the entire DITA project when resources are moved. More about DITA Master files: https://www.oxygenxml.com/doc/versions/22.1/ug-editor/topics/dita-master-files.html.
